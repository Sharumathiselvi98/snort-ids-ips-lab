Introduction:

Snort is a Intrusion Prevention System (IPS) used globally. It keeps an eye on network traffic using a rule-based approach. With its predefined rules, it spots and alerts about suspicious activities in real-time. These rules help it detect various security threats, like unauthorized access or strange behavior on the network. Snort gives network administrators the power to protect their networks from cyber threats, ensuring they stay safe and secure.

Defenection:

    Snort is the foremost Open Source Intrusion Prevention System (IPS) in the world. Snort IPS uses a series of rules that help define malicious network activity and uses those rules to find packets that match against them and generate alerts for users.

Instruction detection system:

    IDS can identify threats but require user assistance to stop them.

Instruction prevention system:

    IPS can identify and block the threats with less user assistance at the detection time.

Installation and configuration:

To install snort in linux. 

sudo apt install snort

At the end of installation you will get a “Packet configuration pop -up” - check the ip address if not change your device id address and hit ok.

Configuration file where users can define the various parameters and rules for snort operations.

Open the Snort configuration file located at etc/snort.conf, and replace the value "any" with the IP address of your device in the line containing "ipvar HOME_NET any".

Save the file,then run the snort.conf file in test mode. This will allow administrators to verify that the configuration file is correctly written and does not contain any errors. 

sudo snort  -T -i enp0s3 -c /etc/snort/snort.conf

Tutorial:

Snort rule structure:

Rule syntax:

alert icmp any any <> any any (msg: “ICMP alert”; reference:CVE-XXX;  sid:1000001; rev:1)
  • msg - message
  • reference - reference
  • sid - rule id
  • rev - revision information

IP Filtering

alert icmp IP_address any <> any any  (msg: "ICMP Packet From "; sid: 100001; rev:1;)

This rule will create an alert for each ICMP packet originating from the 10.0.2.4 IP address.

Filter an IP range

alert icmp IP_address/24 any <> any any (msg: "ICMP Packet Found"; sid: 100001; rev:1;)

This rule will create an alert for each ICMP packet originating from the 10.0.2.4/24 subnet.

Filter multiple IP ranges

alert icmp [IP_address1/24, IP_address2/24] any <> any any (msg: "ICMP Packet Found"; sid: 100001; rev:1;)

This rule will create an alert for each ICMP packet originating from the 10.0.2.4/24 and 10.0.2.5/24 subnets.

Exclude IP addresses

alert icmp !IP_address/24 any <> any any (msg: "ICMP Packet Found"; sid: 100001; rev:1;)

This rule will create an alert for each ICMP packet not originating from the 10.0.2.4/24 subnet.

Port Filtering

alert tcp any any <> any 21 (msg: "FTP Port 21 Command Activity Detected"; sid: 100001; rev:1;)

This rule will create an alert for each TCP packet sent to port 21.

Exclude a specific port

alert tcp any any <> any !21 (msg: "Traffic Activity Without FTP Port 21 Command Channel"; sid: 100001; rev:1;)

This rule will create an alert for each TCP packet not sent to port 21.

Filter a port range

alert tcp any any <> any 1:1024 (msg: "TCP 1-1024 System Port Activity"; sid: 100001; rev:1;)

This rule will create an alert for each TCP packet sent to ports between 1-1024.

Conclusion:

The structure of Snort rules typically consists of the following components: header, rule options, and rule actions. These elements define the conditions for detecting network traffic patterns and specifying actions to take upon detection. In conclusion, Snort rules are essential for customizing intrusion detection and prevention capabilities to protect networks from various security threats.
